** A2. Oracle CLI sanity (expected outcomes documented)
Expected results for current ROM (oos168.sfc):
- Structural smoke: `ok=true` (exit 0). D4 structural_ok=true, D6 meets_min=true.
- Strict readiness: `ok=false` (exit 1). D4 required_rooms_ok=false (0x25 not authored), D3 ok=false (0x32 not authored). This is NOT a regression — it's an authoring gap.
- Preflight: `ok=false` (exit 1). ORACLE_REQUIRED_ROOM_MISSING_COLLISION for 0x25. This is expected.
If structural smoke fails (ok=false in default mode), that IS a regression. Investigate immediately.

* Block B: Dungeon Editor Manual UX (25-35 min)
** B1. Placement limit enforcement
- [ ] Tile object ghost shows normal/near-limit(≥360)/blocked(≥400) colors.
- [ ] Sprite ghost shows normal/near-limit(≥58)/blocked(≥64) colors.
- [ ] Door ghost shows normal/near-limit(≥14)/blocked(≥16) colors.
- [ ] At limit: click placement does NOT add entity. Toast or tooltip confirms block.
- [ ] One below limit: placement succeeds. Entity appears in room.
- [ ] Invalid door placement (center room, far from walls) gives kInvalidPosition error.
- [ ] Placement blocked messages appear, dedup within 1s, and clear after ~2s TTL.
- [ ] Limits match `zelda3/dungeon/dungeon_limits.h` canonical values (regression guard).

** B2. Workbench navigation/visibility
- [ ] Room badge shows `0xHEX (DEC)` format.
- [ ] Copy room ID button copies correct `0x...` value.
- [ ] D6 quick-nav buttons jump correctly: `0xA8, 0xB8, 0xD8, 0xDA`.
- [ ] Custom object overlay toggle visibly affects canvas.
- [ ] Recent rooms list can re-open prior rooms correctly.

* Block C: Overworld + Tile16 UX (20-25 min)
- [ ] Hover tooltip shows tile ID + zoom preview.
- [ ] Jump-to-tile works for valid hex ID.
- [ ] Jump-to-tile ignores invalid ID without crashing.
- [ ] Range filter valid interval behaves correctly.
- [ ] Range filter invalid (`min > max`) shows inline error.
- [ ] Range filter out-of-range input shows inline error.
- [ ] Clear filter restores full interaction.
