# Agent Onboarding Documentation

This directory contains onboarding materials for AI agents joining the yaze development hive.

## Purpose

Agent onboarding docs help new personas understand:
- Project structure and key files
- Development workflows and conventions
- Agent collaboration protocols
- Specialized knowledge areas

## Current Onboarding Docs

- `CLAUDE_AIINF_HANDOFF.md` - Onboarding for AI infrastructure work

## Onboarding Process

### For New Agents

1. **Read Core Documentation**
   - `AGENTS.md` (project root) - Agent coordination requirements
   - `CLAUDE.md` (project root) - Development guidelines
   - `docs/internal/agents/personas.md` - Role definitions
   - `docs/internal/agents/HIVE-SYSTEM.md` - Collaboration system

2. **Review Coordination Board**
   - Check `docs/internal/agents/coordination-board.md` for active work
   - Read recent entries to understand current state
   - Acknowledge any pending requests targeting your persona

3. **Read Persona-Specific Onboarding**
   - Review any handoff docs specific to your assigned role
   - Understand specialized knowledge requirements
   - Note any ongoing initiatives or blockers

4. **First Actions**
   - Introduce yourself on the coordination board
   - Ask questions about unclear areas
   - Take on a small task to familiarize yourself with the codebase

### For Creating Onboarding Docs

When creating a new agent onboarding doc:
1. Include project overview and key responsibilities
2. List critical files and their purposes
3. Document common workflows and conventions
4. Highlight gotchas and known issues
5. Provide examples of typical tasks
6. Link to relevant external resources
