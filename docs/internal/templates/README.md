# Documentation Templates

This directory contains reusable templates for common documentation tasks.

## Available Templates

### Release Checklist Template
- **File:** `release-checklist-template.md`
- **Purpose:** Template for creating release validation checklists
- **Usage:** Copy and customize for each release version
- **Active Checklist:** `../release-checklist.md`

## Using Templates

1. Copy the template file to the appropriate location
2. Rename with specific context (e.g., `release-checklist-v0.4.0.md`)
3. Fill in version-specific information
4. Update dates and assignees
5. Track progress through completion

## Adding New Templates

When creating a new template:
1. Document the template's purpose and use case
2. Include clear placeholders (e.g., `[VERSION]`, `[DATE]`)
3. Add usage instructions in comments
4. Update this README with the new template

## Template Guidelines

- Keep templates generic and reusable
- Use markdown format for consistency
- Include sections that commonly repeat across documents
- Avoid project-specific details (those go in active documents)
- Version control templates for tracking improvements
